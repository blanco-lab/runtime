# ADR-0004: Capability / Backend Separation

- Estado: ACEPTADO
- Fecha: 2026-07-18
- Decisores: Blanco (Director), Atlas (Chief Architect)
- Contexto de origen: AR-002 (primera Capability real de Runtime)

## Contexto

Runtime debe actuar sobre el sistema (reproducir música, enviar correo,
etc.) sin acoplarse a aplicaciones concretas. Si Runtime dependiera
directamente de Spotify, cambiar a mpv o Jellyfin obligaría a tocar el
pipeline. Eso rompe el principio de independencia de proveedor de la
Constitución.

## Decisión

Runtime nunca depende de aplicaciones concretas. Runtime solo conoce
**Capabilities** (capacidades abstractas). Las aplicaciones reales son
**Backends** intercambiables.

    Capability: music_player      (lo que Runtime conoce)
        └── Backend: spotify_player   (implementación concreta, hoy)
        └── Backend: mpv              (futuro)
        └── Backend: jellyfin         (futuro)

Reglas:

1. La Capability define una interfaz estable
   (`play/pause/resume/next/previous/volume`).
2. La Capability NO elige backend. El backend se inyecta.
   Hoy se instancia `spotify_player` directamente; mañana un
   **BackendFactory** lo resolverá por configuración / descubrimiento /
   perfil de usuario, sin tocar la Capability ni el pipeline.
3. Cambiar de backend nunca debe afectar al pipeline.

## Consecuencias

- El mismo diseño se reutiliza para correo, calendario, WhatsApp,
  navegador, SuiteCRM y cualquier integración futura.
- El pipeline (Intent → Action → Safety → Executor) permanece agnóstico.
- Coste hoy: una capa de indirección mínima. Beneficio: desacoplamiento
  total aplicación ↔ Runtime.

## Estado de implementación (Fase 1)

- `src/core/capabilities/backend.py`: interfaz `Backend` (ABC).
- `src/core/capabilities/music_player/__init__.py`: Capability
  `MusicPlayer`, backend inyectable (hoy spotify_player por defecto).
- `src/core/capabilities/music_player/backends/spotify_player.py`:
  Backend real contra el CLI `spotify_player`.
- BackendFactory: NO implementado aún (preparada la separación).
