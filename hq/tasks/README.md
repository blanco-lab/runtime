# Runtime HQ - Task System

## Objetivo

Las tareas representan el trabajo oficial del proyecto.

Toda tarea debe existir como un documento independiente.

Nunca se trabajará sobre una tarea que no exista en este directorio.

---

## Convención de nombres

TASK-0001.md

TASK-0002.md

TASK-0003.md

...

---

## Estados

TODO

IN_PROGRESS

BLOCKED

REVIEW

DONE

CANCELLED

---

## Estructura

Cada tarea deberá contener:

- Identificador
- Título
- Responsable
- Estado
- Prioridad
- Descripción
- Resultado esperado
- Resultado obtenido
- Historial

---

## Regla

Una tarea no existe oficialmente hasta que tenga su propio archivo.
