# ¿Qué es Runtime?

Runtime es una plataforma de orquestación conversacional para sistemas operativos.

Su propósito es permitir que una persona interactúe con su ordenador mediante lenguaje natural, transformando intenciones en acciones seguras, estructuradas y auditables.

Runtime no es un chatbot.

Runtime no es un asistente virtual.

Runtime no es un intérprete de comandos.

Runtime es una capa de orquestación situada entre el usuario y el sistema operativo.

---

# Nuestra misión

Construir la mejor plataforma abierta para controlar cualquier sistema operativo mediante lenguaje natural.

La plataforma deberá ser:

- segura;
- extensible;
- independiente del proveedor de IA;
- multiplataforma;
- basada en herramientas;
- preparada para evolucionar durante muchos años.

---

# El problema

Los sistemas operativos modernos siguen obligando al usuario a conocer cómo realizar una tarea.

Runtime pretende invertir esa relación.

El usuario expresa una intención.

La plataforma decide cómo satisfacerla utilizando herramientas especializadas.

---

# Nuestra filosofía

Las decisiones del proyecto estarán guiadas por estos principios.

## 1. Seguridad

La seguridad siempre tendrá prioridad sobre la comodidad.

---

## 2. Arquitectura

La arquitectura tendrá prioridad sobre la velocidad de desarrollo.

---

## 3. Transparencia

Toda acción realizada por Runtime deberá poder explicarse.

---

## 4. Modularidad

Todo componente deberá poder sustituirse sin afectar al resto del sistema.

---

## 5. Independencia

Runtime nunca dependerá de un proveedor concreto de inteligencia artificial.

---

## 6. Calidad

La documentación, las pruebas y el código forman parte del mismo producto.

---

# Qué NO es Runtime

Runtime no pretende ser:

- un chatbot;
- un reemplazo del sistema operativo;
- un intérprete de Bash con IA;
- una colección de scripts;
- una automatización puntual.

---

# Visión

Aspiramos a crear una plataforma abierta capaz de convertirse en un estándar para la interacción conversacional con sistemas operativos.

Nuestro objetivo no es construir una aplicación.

Nuestro objetivo es construir una infraestructura.
